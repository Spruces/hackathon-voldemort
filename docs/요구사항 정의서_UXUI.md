## A. 화면 흐름 · 플로우

### A-1. 접근 플로우 — 비밀번호로 2개 모드 분기

```
                                  ┌ 임원용 6자리 입력 ──▶ ① 임원(조회) 모드
전용 URL 접속 / QR 스캔  →  비번 입력 ┤
                                  └ 사장님 전용 비번 입력 ─▶ ② 사장님(편집) 모드
   인증 성공 시 7일간 세션 유지 (재방문 시 재입력 X)
```

**① 임원(조회) 모드** — 편집 버튼 없음, 🔒 비공개 항목(자유 메모) 안 보임, 📢 공개 항목만.
**② 사장님(편집) 모드** — 조회 + 편집, 🔒 비공개 항목 열람·수정, 관리 기능.
> 같은 레이아웃, **모드에 따라 편집 UI·🔒 노출만 달라짐**. 모드 판정은 서버에서.
> 사장님 모드 한정 **"임원 모드 미리보기" 토글**(FR-08a) — 임원 화면을 그대로 시뮬레이션.

### A-2. 업종 탭 필터 인터랙션

```
전체 보기(기본)  →  업종 탭 선택  →  해당 업종만 표시  →  카드 클릭 → 상세
```
- 탭은 가로 스크롤. `전체` + 7종 = 8개.

### A-3. 사장님 편집 플로우 (FR-10/11)

```
Owner 로그인 → 식당 상세 → [편집] → 입력 → 저장
                                   ├ 메모(자유) ......... 🔒 Owner 전용 (임원 노출 X)
                                   └ 소개/영업시간/주차 .. 📢 임원 노출
```

### A-4. 신규 맛집 검색 플로우 (FR-12, 조회 전용)

```
Owner → 지도 검색창에 식당명/지역 입력 → 지도 API 결과 표시 → 확인만 (DB 저장 없음)
```

## D. VIP 디자인 방향성

- **색상**: 다크 배경 + 골드 액센트
- **타이포**: 세리프 타이틀 + 산세리프 본문
- **레이아웃**: 넓은 여백, 절제된 정보 밀도
- **인터랙션**: 과한 애니메이션 지양. 호버/전환 효과 최소화. 정보 신뢰도·고급감 우선
- **정보 계층**: 리스트 카드(식당명·업종·위치) → 상세(전체 정보 + 지도 + 사장님 메모)
