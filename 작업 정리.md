## 3. 요구사항 정의서 핵심 보완 포인트 (데이터 기반)

1. **FR-01 업종 필터** — 종류 34종은 그대로 탭 사용 불가 → 정규화 매핑 테이블 필수
2. **FR-02/03 카드·상세** — 별점·리뷰·예약·메뉴·이미지 0건 → 빈 상태/스켈레톤/플레이스홀더 처리
3. **FR-09 Bedrock 보강** — "결측 보완"이 아니라 **상세 진입 시 매번 호출이 기본** → 캐싱(TTL)·fallback 필요
4. **메모 컬럼 재정의** — "Owner 비공개"라지만 실제론 공개 설명·영업시간(27건)·예약방식(5건)·추천인 혼재 → `public_desc`/`hours`/`internal_memo` 분해
5. **신규 데이터 모델** — 그룹·비밀번호·식당매핑 소스에 전무 → Group / GroupRestaurant / Enrichment / Session 신규 설계

## 4. 프로덕션 전환 권고 (요약)

> 해커톤 데모 ≠ 제품. 차이는 **신뢰·보안·운영**. 우선순위 순.

| 순위 | 영역 | 핵심 |
|:---:|------|------|
| **P0** | AI 데이터 신뢰성 | 별점·리뷰·메뉴가 0건 → LLM 생성값을 사실처럼 노출하는 구조. **출처/신뢰도 라벨**, 검증 가능 소스 우선, 불확실 시 빈칸 유지 |
| **P0** | 접근 보안 | 4자리(1만 조합)는 데모용 → **무차별 대입 방지·해시 저장·토큰 엔트로피·세션 보안·감사 로그** |
| **P1** | 데이터 운영 | 엑셀 1회 적재 탈피 → CRUD UI/재업로드, 정규화 매핑 운영화, 임포트 검증 |
| **P1** | 운영 인프라 | 환경 분리·CI/CD·백업·모니터링·Bedrock 비용/실패율 대시보드 |
| **P1~P2** | 콘텐츠·UX | 이미지 라이선스(무단 크롤링 리스크), 국제화(한/영), 상태 디자인 |
| **P2** | 제품 구조 | 멀티테넌시(다국가·SaaS), 사용 분석 큐레이션 |
