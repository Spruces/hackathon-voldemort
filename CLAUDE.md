# CLAUDE.md — 식당 DB 대시보드 (개발 가이드)

> 본 파일은 Claude Code가 매 대화마다 자동으로 읽는 프로젝트 안내문이다.

---

## 1. 프로젝트 컨텍스트

- **상황**: 해커톤 진행 중. 팀원이 **Git 저장소를 통해** 같은 폴더의 문서·코드를 공유하며 작업한다.
- **주제**: 사장님 식당 DB(228건)를 VIP 임원에게 전용 URL+6자리 비번으로 제공하는 모바일 우선 대시보드.
- **단일원천(SSOT)**: [`docs/요구사항 정의서.md`](docs/요구사항%20정의서.md) — 모든 기능·정책·데이터 규칙의 최종 정본. 코드 구현 시 이 문서를 기준으로 삼는다.
- **프로젝트 구조**:
  ```
  /
  ├── CLAUDE.md              ← 이 파일 (개발 가이드)
  ├── docs/                  ← 기획 문서 (요구사항 정의서, 작업 정리 등)
  ├── data/                  ← 원본 데이터 (xlsx, csv, json)
  ├── mockup/                ← UI 목업
  └── (소스코드 — 아래 구현 가이드 참고)
  ```
- **핵심 결정사항**: Bedrock(LLM) → **네이버/카카오 지도 공식 API**, 사장님 편집(FR-10/11), 크롤링 미채택, 업종 탭 7종, 자유 메모 Owner 전용+노출 배지(🔒/📢), 단일 URL+6자리 비번 2종 모드 분기, 세션 7일.

## 2. 역할

- **이 세션에서 Claude는 개발자 역할**이다. 코드를 직접 작성·구현한다.
- 기획/요구사항은 `docs/요구사항 정의서.md`(단일원천)를 참조한다.
- 구현 상세는 `docs/IMPLEMENTATION.md`를 참조한다.

## 4. 디자인 방향

- **타겟**: 50대 남성(사장님) + C-Level 임원 → 사용 편의 + 전문적 고급감 동시 충족
- **모바일 우선** + 데스크탑 반응형
- **색상**: 다크 배경 + 골드 액센트
- **타이포**: 큰 텍스트, 충분한 터치 영역, 가독성 우선
- **레이아웃**: 넓은 여백, 절제된 정보 밀도, 과한 애니메이션 지양
- **목표**: 사장단끼리 보여줘도 부끄럽지 않은 전문적 프로덕트 느낌
